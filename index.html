On-device PDP assistant

An assistant that reads the product page it's sitting on — and answers with Chrome's local model, not a cloud one.

Paste one snippet into DevTools Console on any product page. It extracts what's actually on the page (JSON-LD, price, specs, description) and answers shopper questions using Chrome's built-in Gemini Nano. Nothing — page content or questions — leaves the browser.

Run it on a real PDP

Open the target product page, open DevTools Console (⌥⌘J / Ctrl+Shift+J), paste this, press Enter:

(async () => { const url = "REPLACE_WITH_YOUR_DEPLOY_URL/koctas-nano-assistant.js"; try { const res = await fetch(url, { cache: "no-store" }); if (!res.ok) throw new Error("HTTP " + res.status); (0, eval)(await res.text()); } catch (e) { console.error("[PDP Nano Assistant] couldn't load hosted bundle:", e); console.warn("If this page's CSP blocks the fetch, paste the full bundle.js source directly into Console instead."); } })();

Before it'll answer anything

This needs Chrome's on-device model, which is still flag-gated for regular web pages as of mid-2026. In a new tab, set both of these to Enabled, then relaunch Chrome:

chrome://flags/#optimization-guide-on-device-model chrome://flags/#prompt-api-for-gemini-nano
  • Chrome 138 or newer, desktop
  • First run downloads the model (a few GB, one-time, shared across all sites) — the widget shows a progress bar for this
  • No flags, no download, no LanguageModel global → the widget explains that plainly instead of pretending to work
runs entirely on-device · no product data or questions leave the browser · no cloud LLM in this path
